# 主线与支线设计指南

> 本指南对应评分体系"剧情与结构"维度，将"审什么"转化为"怎么写"。

---

## 一、核心原则

1. **主线是脊椎**：所有元素都必须服务于主线，主线断裂则故事散架
2. **支线是血肉**：支线丰富主线、不能喧宾夺主
3. **伏笔是种子**：每一个伏笔都必须回收，遗忘的伏笔是烂尾的前兆
4. **转折是引擎**：好的转折让读者"意外又信服"
5. **情感弧线是灵魂**：角色必须变化，否则只是"经历事件"而非"经历故事"

---

## 二、主线设计方法

### 2.1 构建核心冲突

**【目标】**：明确故事的核心驱动力，让读者知道主角"为什么必须行动"

**【方法】**：核心冲突 = 主角的强烈欲望 + 巨大的实现障碍

```
冲突设计公式：
核心冲突 = 主体(Who) × 欲望(What) × 障碍(Why)
- 主体：有辨识度的主角
- 欲望：具体、可量化
- 障碍：足够大但可克服
```

**【正例特征】**：
- 《斗破苍穹》：萧炎（主体）× 重振家族（欲望）× 资质被废+退婚之辱（障碍）
- 《三体》：人类文明（主体）× 生存下去（欲望）× 三体文明入侵（障碍）

**【反例特征】**：
- 主角想要"幸福"，障碍是"生活不易"——太模糊，无推动力
- 主角想要"拯救世界"，但没有任何具体原因——缺乏情感锚点

**【AI指令模板】**：

```
请为主角设计核心冲突：
1. 主角的身份/背景：[描述]
2. 主角最想要什么（必须是具体的、可观察的）：[描述]
3. 为什么他不能轻易得到：[列出至少2个重大障碍]
4. 这个冲突如何影响主角的内心：[描述内心挣扎]

输出格式：
- 核心冲突一句话概括：[80字以内]
- 欲望：[具体描述]
- 障碍清单：[按重要性排序]
- 冲突等级：世界级/社会级/个人级
```

### 2.2 设计情感弧线

**【目标】**：让主角在故事中发生变化，变化是可信的、渐进的

**【方法】**：情感弧线 = 起点状态 → 试炼 → 转折点 → 新状态

```
情感弧线四要素：
1. 起点：主角的"默认值"（性格、信念、价值观）
2. 试炼：挑战默认值的事件
3. 转折：主角做出改变的关键时刻
4. 新状态：主角的变化结果
```

**【正例特征】**（《活着》福贵）：
- 起点：纨绔子弟
- 试炼：家产输光、战乱被抓
- 转折：妻子去世、女儿去世
- 新状态：活着本身成为意义

**【反例特征】**：
- 主角经历重大事件但毫无反应
- 角色变化是"顿悟"而非渐变
- 所有角色一起变，没有主次

**【AI指令模板】**：

```
请为主角设计情感弧线：

【起点状态】
- 性格特征：[列出3个核心特征]
- 核心信念：[主角相信什么]
- 价值观：[主角看重什么]

【试炼事件】（至少3个，按顺序）
1. [第一个试炼] → 主角如何应对：[描述]
2. [第二个试炼] → 主角开始动摇：[描述]
3. [第三个试炼] → 主角被迫改变：[描述]

【转折点】
- 触发事件：[描述]
- 转折时刻的内心独白：[主角想什么]
- 决定：[主角做了什么决定]

【新状态】
- 性格变化：[列出变化前后对比]
- 核心信念变化：[从X变为Y]
- 象征这种变化的物品/动作：[描述]

【变化可信度检查】
- [ ] 变化是由事件驱动而非强行安排
- [ ] 变化有足够的铺垫
- [ ] 变化是不可逆的
```

### 2.3 起承转合规划

**【目标】**：确保故事结构完整，有头有尾有高潮

**【方法】**：起承转合 = 建置 → 上升 → 高潮 → 解决

```
四幕结构：
┌─────────────────────────────────────────┐
│ 第一幕·建置（25%）                       │
│ - 展示主角的"默认值"生活                │
│ - 触发事件打破平衡                       │
│ - 主角接受挑战                           │
├─────────────────────────────────────────┤
│ 第二幕·上升（25%）                      │
│ - 主角尝试解决问题                       │
│ - 遭遇挫折                               │
│ - 建立支线/配角关系                      │
├─────────────────────────────────────────┤
│ 第三幕·高潮（25%）                      │
│ - 所有问题汇聚                           │
│ - 最大冲突爆发                           │
│ - 主角面临最终选择                       │
├─────────────────────────────────────────┤
│ 第四幕·解决（25%）                      │
│ - 冲突解决                               │
│ - 角色变化落定                           │
│ - 回应开头                               │
└─────────────────────────────────────────┘
```

**【AI指令模板】**：

```
请为[小说类型/篇幅]设计起承转合：

【第一幕·建置】
- 开头场景：[描述，200字]
- 主角现状：[描述日常]
- 触发事件：[什么打破平衡]
- 主角决定：[主角接受/拒绝挑战]
- 字数比例：__%

【第二幕·上升】
- 第一个目标：[描述]
- 第一个障碍：[描述]
- 第一个高潮：[描述]
- 支线引入：[列出支线]
- 字数比例：__%

【第三幕·高潮】
- 中点转折：[从积极转消极或反之]
- 最大障碍：[描述]
- 主角的困境：[内心+外部双重压力]
- 最终选择：[主角的抉择]
- 字数比例：__%

【第四幕·解决】
- 核心问题解决：[描述]
- 呼应开头：[首尾呼应设计]
- 结局基调：[余韵/圆满/留白]
- 字数比例：__%
```

## 三、支线设计方法

### 3.1 支线服务主线

**【目标】**：每一条支线都必须有意义，服务于主线

**【方法】**：支线功能分类

| 支线类型 | 服务主线方式 | 举例 |
|---------|-------------|------|
| **人物支线** | 丰富主角/揭示配角 | 主角的亲情线揭示其软肋 |
| **主题支线** | 深化核心主题 | 主线打怪，支线探讨"力量代价" |
| **信息支线** | 补充世界观/背景 | 通过支线人物揭示世界规则 |
| **情感支线** | 调节节奏/增加层次 | 高强度主线间插入温情支线 |

**【正例特征】**（《冰与火之歌》小恶魔支线）：
- 表面是提利昂的个人命运
- 实质是揭示"弑父者"真相
- 最终影响主线剧情走向

**【反例特征】**：
- 支线与主线完全无关，删除后毫无影响
- 支线太多，主线被淹没
- 支线没有收束，烂尾

**【AI指令模板】**：

```
请为以下主线设计支线：

【主线】
- 核心冲突：[描述]
- 当前阶段：[建置/上升/高潮/解决]
- 主角当前困境：[描述]

【请设计3条支线】
支线1：
- 类型：[人物/主题/信息/情感]
- 内容：[描述]
- 服务主线的方式：[描述]
- 与主线的交叉点：[描述]
- 是否需要回收：[是/否]

支线2：同上
支线3：同上

【支线节奏规划】
- 支线1首次出现：第X章
- 支线2首次出现：第X章
- 支线3首次出现：第X章
- 支线收束时间：第X章
```

### 3.2 支线展开与收束节奏

**【目标】**：支线有引入、有发展、有回收，不能半途而废

**【方法】**："钩子-发展-回收"三段式

```
支线节奏：
引入（Hook）→ 发展（Development）→ 回收（Resolution）

- 引入：当XX事件发生，支线开启
- 发展：主角在主线间隙推进支线
- 回收：支线高潮与主线高潮呼应
```

**【AI指令模板】**：

```
请规划以下支线的节奏：

支线名称：[名称]
支线核心：[描述支线的核心事件]

【引入】
- 触发事件：[描述]
- 触发时机：[第X章/主线发展到XX阶段]
- 引发好奇：[1句话钩子]

【发展】（至少3个阶段）
阶段1：[描述] → 结果：[描述]
阶段2：[描述] → 结果：[描述]
阶段3：[描述] → 结果：[描述]

【回收】
- 回收时机：[第X章，建议与主线高潮呼应]
- 回收方式：[揭示真相/完成使命/影响主线]
- 情感落点：[读者此刻应感受到什么]

【与主线的关系】
- 如何影响主线：[描述]
- 是否改变主线走向：[是/否]
```

## 四、伏笔系统设计

### 4.1 埋伏笔的方法

**【目标】**：让伏笔自然融入，不引起读者注意，但在回收时让读者恍然大悟

**【方法】**：伏笔融入场景描写的三种方式

| 融入方式 | 操作方法 | 适用场景 |
|---------|---------|---------|
| **物品伏笔** | 在场景中自然提及某物品，该物品后成为关键道具 | 需要实体线索时 |
| **语言伏笔** | 角色随口一句话，后成关键信息 | 需要信息揭示时 |
| **动作伏笔** | 角色的习惯性动作，后揭示原因 | 需要情感揭示时 |

**【正例特征】**（《盗墓笔记》）：
- 开篇提到"战国帛书"，贯穿全书
- 张起灵"你不懂"反复出现，暗示身世之谜

**【反例特征】**：
- 伏笔太明显，读者一眼看穿
- 伏笔与环境割裂，"凭空出现"
- 同一场景埋伏笔太多，重点不清

**【AI指令模板】**：

```
请为[小说类型]设计伏笔系统：

【需要埋伏笔的关键信息】
1. [信息1] → 建议埋设时机：[早期/中期/后期]
2. [信息2] → 建议埋设时机：[早期/中期/后期]
3. [信息3] → 建议埋设时机：[早期/中期/后期]

【伏笔设计】（每个信息设计一个伏笔）
伏笔1：
- 信息：[要回收的信息]
- 伏笔类型：[物品/语言/动作]
- 埋设方式：[详细描述如何自然出现]
- 埋设位置：[场景/时机]
- 回收方式：[如何让读者恍然大悟]

伏笔2：同上
伏笔3：同上

【伏笔清单】（按章节排列）
| 章节 | 伏笔类型 | 伏笔描述 | 回收章节 |
|------|---------|---------|---------|
| 第X章 | 物品 | 描述 | 第Y章 |
```

### 4.2 伏笔回收方法

**【目标】**：伏笔回收时让读者感到"原来如此"而非"这也太巧了"

**【方法】**：回收三要素

```
伏笔回收公式：
情感冲击（读者情绪峰值）+ 逻辑连接（前文依据）+ 延展空间（引发新问题）

- 情感冲击：回收时是正面还是负面情绪？
- 逻辑连接：之前埋下的线索是否清晰？
- 延展空间：回收后是否引发新的悬念？
```

**【AI指令模板】**：

```
请为以下伏笔设计回收：

伏笔描述：[伏笔内容]
伏笔埋设位置：[第X章]

【回收设计】
- 回收时机：[第Y章，此时主角/读者处于什么状态]
- 触发方式：[什么事件触发伏笔回收]
- 回收内容：[揭示的信息]

【情感设计】
- 读者预期情绪：[如惊讶/悲伤/恍然大悟]
- 情感对比：[如果与读者预期相反会更有冲击力]
- 情感峰值设计：[最高点在哪一刻]

【逻辑设计】
- 前文线索回顾：[列出之前埋下的所有相关线索]
- 连接方式：[如何将线索串联起来]
- 是否需要额外解释：[读者可能需要什么信息才能理解]

【延展设计】
- 回收后引发的新问题：[1-2个]
- 是否引出新的主线：[是/否]

【伏笔回收检查】
- [ ] 伏笔足够隐蔽但可被发现
- [ ] 回收有足够情感冲击力
- [ ] 逻辑链完整
- [ ] 不需要额外解释（除非有意留白）
```

## 五、转折设计方法

### 5.1 让转折"意料之外情理之中"

**【目标】**：转折让读者惊讶但不觉得被骗

**【方法】**：转折前必须有"线索"，但线索要伪装成"正常"

```
转折设计公式：
转折 = 必然性（已有铺垫）+ 偶然性（最后一刻发生）+ 合理性（符合世界观）

- 必然性：这个结果必须有前文支撑
- 偶然性：具体如何发生可以有偶然因素
- 合理性：不能违反世界观规则
```

**【正例特征】**（《看不见的客人》）：
- 每一个转折都有证据支撑
- 观众回头看会发现"线索早就给出"

**【反例特征】**：
- "天降神兵"解决问题
- "主角突然想起"有这个能力
- "反派恰好"犯了一个致命错误

**【AI指令模板】**：

```
请为以下情节设计转折：

【原始情节】
- 场景：[描述]
- 预期走向：[原本会如何发展]

【转折设计】
- 转折类型：[身份揭示/力量觉醒/外部介入/内心转变]
- 转折结果：[具体是什么变化]
- 发生时机：[什么时候揭晓]

【铺垫设计】（必须让转折"合理"）
铺垫1：
- 内容：[描述]
- 出现位置：[第X章]
- 伪装方式：[如何让这看起来像普通情节]

铺垫2：[如有需要]

【合理性检查】
- [ ] 转折不违反世界观规则
- [ ] 主角/读者有足够信息理解转折
- [ ] 转折后故事仍然可以继续

【意外性检查】
- [ ] 读者不会在转折前猜到这个结果
- [ ] 转折足够"震撼"但不是"惊吓"
```

### 5.2 转折与高潮设计

**【目标】**：让高潮成为全故事的转折点，并服务于情感弧线

**【方法】**：高潮 = 主线冲突爆发 + 情感弧线落点 + 价值观抉择

```
高潮三要素：
1. 主线冲突爆发：所有问题汇聚，必须有一个结果
2. 情感弧线落点：主角的变化在这一刻体现
3. 价值观抉择：主角做出什么选择，定义了他是什么人
```

**【AI指令模板】**：

```
请设计高潮场景：

【当前情节状态】
- 主线冲突：[描述]
- 主角情感状态：[描述]
- 当前困境：[描述]

【高潮设计】
- 高潮事件：[描述具体事件]
- 核心冲突爆发点：[哪一刻所有矛盾爆发]
- 主角的最终选择：[描述]

【情感弧线落点】
- 主角的变化：[从X到Y]
- 变化在这一刻的体现：[通过什么动作/决定体现]

【价值观表达】
- 主角选择背后的价值观：[主角相信什么]
- 与开头的呼应：[开头主角会如何选择，与现在对比]

【张力节奏】
- 高潮前的铺垫：[最后蓄力阶段]
- 高潮峰值：[最高点在哪]
- 高潮后落点：[如何从高潮过渡到结局]

【感官设计】
- 视觉冲击：[读者"看到"什么]
- 听觉冲击：[读者"听到"什么]
- 情感冲击：[读者"感受到"什么]
```

## 六、结尾设计方法

### 6.1 让结尾有力

**【目标】**：结尾让读者有完整感，并留下余韵

**【方法】**：结尾四要素

```
结尾公式：
回应（回应开头）+ 落点（情感落定）+ 余韵（引发思考）+ 象征（用具体结束）

- 回应：呼应开头，形成闭环
- 落点：主角状态落定
- 余韵：引发读者思考
- 象征：用具体动作/物品结束
```

**【正例特征】**（《活着》结尾）：
- "我看到广阔的土地袒露着结实的胸膛，那是召唤的姿态"
- 回应开头"我是凤霞"，形成生命循环的象征

**【AI指令模板】**：

```
请设计结尾：

【故事整体】
- 开头状态：[描述]
- 主角最终状态：[描述]
- 核心主题：[小说想表达什么]

【结尾设计】
- 结局类型：[圆满/悲剧/留白/循环]
- 具体场景：[描述最后场景]

【呼应设计】
- 与开头的呼应点：[描述]
- 呼应方式：[场景/意象/对话/状态]

【情感落点】
- 读者最终感受：[如释重负/悲伤/平静/思考]
- 主角最终状态：[描述]

【余韵设计】
- 留白内容：[什么没有说完]
- 引发思考的问题：[1-2个]
- 象征设计：[用何物/动作/意象收尾]

【检查清单】
- [ ] 回应了开头
- [ ] 主角弧线完整
- [ ] 核心问题解决
- [ ] 留有适当余韵
- [ ] 结尾不仓促
```

## 七、长篇vs短篇结构差异

### 7.1 短篇结构（≤2万字）

| 要素 | 要求 | 说明 |
|------|------|------|
| 主线 | 单一 | 一条主线贯穿，不设支线 |
| 冲突 | 集中 | 围绕一个核心冲突展开 |
| 人物 | 精简 | 主角+1-2个关键人物 |
| 伏笔 | 少量 | 1-2个核心伏笔即可 |
| 节奏 | 紧凑 | 快速进入，快速推进，快速落幕 |

**【短篇AI指令模板】**：

```
请写一个[字数]字的短篇：

【核心要素】
- 类型：[类型]
- 主题：[一句话主题]
- 视角：[第一人称/第三人称]

【情节设计】
- 开头（10%）：[触发事件，100字]
- 发展（40%）：[冲突展开，400字]
- 高潮（30%）：[转折/解决，300字]
- 结尾（20%）：[余韵，200字]

【伏笔】
- 伏笔1：[描述] → 回收点：[描述]

【结尾】
- 类型：[圆满/悲剧/留白/意外]
- 具体设计：[描述]
```

### 7.2 长篇结构（>8万字）

| 要素 | 要求 | 说明 |
|------|------|------|
| 主线 | 清晰但可发展 | 主线明确，中途可微调 |
| 支线 | 丰富 | 2-5条主要支线 |
| 人物 | 群像 | 主角+配角+反派，形成关系网 |
| 伏笔 | 系统化 | 建立伏笔跟踪表 |
| 节奏 | 起伏 | 大高潮套小高潮 |

**【长篇AI指令模板】**：

```
请为长篇小说设计结构：

【基本信息】
- 篇幅：[预计字数]
- 类型：[类型]
- 预计章节：[X章]

【整体结构】
- 主线：[描述]
- 支线清单：[列出主要支线]

【章节规划】
第一阶段（第1-__章）：
- 阶段主题：[描述]
- 主要事件：[列出]
- 情感弧线：[主角当前状态]

第二阶段（第__-__章）：
[同上]

第三阶段（第__-__章）：
[同上]

第四阶段（第__-__章）：
[同上]

【伏笔系统】
| 伏笔 | 埋设章节 | 回收章节 | 备注 |
|------|---------|---------|------|
| 伏笔1 | 第X章 | 第Y章 | |
```

## 八、常见问题与修复

### 问题1：情节太拖沓

**表现**：读者感觉"怎么还没完"

**修复方法**：
1. 检查每章是否有明确目的
2. 删除与主线无关的描写
3. 合并重复的情节节点
4. 增加障碍和挫折

### 问题2：转折太突兀

**表现**：读者觉得"这也太巧了"

**修复方法**：
1. 回顾前文，找出可以改为"伏笔"的元素
2. 将"突然发生"改为"意料之外情理之中"
3. 增加过渡场景

### 问题3：伏笔遗忘

**表现**：开头埋下的伏笔完全没有回收

**修复方法**：
1. 建立伏笔跟踪表
2. 每写完10%内容检查伏笔状态
3. 确保伏笔回收服务于当前情节

### 问题4：支线喧宾夺主

**表现**：配角戏份比主角还多

**修复方法**：
1. 检查每个配角的功能
2. 限制支线章节数
3. 确保支线回归主线

### 问题5：结尾太仓促

**表现**：高潮后立刻结束，读者感觉"没落地"

**修复方法**：
1. 高潮后增加"落点"场景
2. 给主角和关键人物一个收尾镜头
3. 增加结尾的象征性细节

## 九、自检清单

### 9.1 主线自检

- [ ] 核心冲突是否清晰？
- [ ] 主角的欲望是否具体可量化？
- [ ] 障碍是否足够大但可克服？
- [ ] 情感弧线是否完整？
- [ ] 起承转合是否完整？

### 9.2 支线自检

- [ ] 每条支线是否都有服务主线的方式？
- [ ] 支线是否有引入、发展、回收？
- [ ] 支线是否喧宾夺主？
- [ ] 支线是否都回收了？

### 9.3 伏笔自检

- [ ] 关键伏笔是否有记录？
- [ ] 伏笔是否太明显？
- [ ] 伏笔回收是否有情感冲击力？
- [ ] 是否有遗忘的伏笔？

### 9.4 转折自检

- [ ] 转折是否有铺垫？
- [ ] 转折是否合理（不违反世界观）？
- [ ] 转折是否意外但可信？
- [ ] 高潮是否有力？

### 9.5 结尾自检

- [ ] 是否呼应开头？
- [ ] 主角状态是否落定？
- [ ] 是否有余韵？
- [ ] 是否太仓促？
