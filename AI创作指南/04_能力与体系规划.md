# 能力与体系规划指南

> 本指南综合评分体系中科幻/奇幻/玄幻特有审核项，包括：金手指设计、能力体系设计、战斗设计等。

---

## 一、核心原则

1. **限制即魅力**：没有限制的能力是无聊的，代价让能力更有价值
2. **体系即节奏**：清晰的能力体系是升级节奏的基础
3. **展示即战斗**：能力通过对抗才能展现
4. **金手指要平衡**：足够强让读者爽，但不能强到失去挑战
5. **战力有天花板**：没有天花板就会失去紧张感

## 二、科幻技术体系设计

### 2.1 技术体系设计公式

**【目标】**：让科幻技术可信、有深度、能驱动剧情

**【方法】**：技术体系 = 原理 + 能力 + 限制 + 影响

```
科幻技术四要素：

┌─────────────────────────────────────────┐
│ 原理层：技术的基础                        │
│ - 基于什么科学原理                       │
│ - 从已知到未知的推演                     │
│ - 需要什么假设                           │
├─────────────────────────────────────────┤
│ 能力层：技术能做什么                     │
│ - 核心功能                               │
│ - 扩展功能                               │
│ - 创新功能（独特之处）                   │
├─────────────────────────────────────────┤
│ 限制层：技术不能做什么                   │
│ - 物理限制                               │
│ - 能量限制                               │
│ - 知识限制                               │
├─────────────────────────────────────────┤
│ 影响层：技术改变什么                     │
│ - 个人层面                               │
│ - 社会层面                               │
│ - 世界层面                               │
└─────────────────────────────────────────┘
```

**【正例特征】**（《三体》二向箔）：
- 原理：基于弦理论的降维攻击
- 能力：把三维变成二维
- 限制：不可逆转，无法防御
- 影响：太阳系毁灭

## 三、魔法体系设计

### 3.1 魔法体系核心要素

**【目标】**：让魔法有逻辑、有代价、有层次

**【方法】**：魔法体系 = 魔力来源 + 施法方式 + 咒语/动作 + 代价限制

```
魔法体系四要素：

1. 魔力来源：
   - 外部：自然元素/神灵/他人
   - 内部：生命力/精神力/意志力
   - 特殊：灵魂/血脉/契约

2. 施法方式：
   - 咒语：需要念诵
   - 动作：需要手势/仪式
   - 媒介：需要物品/材料
   - 天赋：天生能力

3. 咒语/动作：
   - 简洁咒语：快速但弱
   - 复杂咒语：强大但慢
   - 禁忌咒语：代价极大

4. 代价限制：
   - 即时代价：消耗什么
   - 累积代价：长期后果
   - 禁忌代价：绝对不能做的事
```

**【正例特征】**（《诡秘之主》）：
- 魔力来源：扮演自己、收集锚点
- 施法方式：仪式+咒语+材料
- 代价限制：失控风险、序列代价
- 独特机制：扮演法

### 3.2 修炼体系设计


## 四、金手指设计

### 4.1 金手指设计原则

**【目标】**：让金手指足够有趣，但不能强到没有挑战

**【方法】**：金手指 = 能力 + 限制 + 代价 + 成长

```
金手指设计四原则：

1. 有趣性：
   - 与众不同的能力
   - 可以被主角"玩出花"的能力
   - 让读者期待"主角怎么用"

2. 限制性：
   - 使用次数限制
   - 使用条件限制
   - 使用范围限制

3. 代价性：
   - 身体代价
   - 精神代价
   - 社会代价

4. 成长性：
   - 能力可以升级
   - 升级需要努力
   - 升级带来新问题
```

**【正例特征】**（《诡秘之主》扮演法）：
- 有趣：需要扮演才能升级，方式独特
- 限制：必须真正成为那个角色
- 代价：扮演不当会失控
- 成长：不同序列有不同能力

**【反例特征】**：
- 金手指太强大，主角没有成长空间
- 金手指规则随意改变
- 金手指与世界观脱节

### 4.2 金手指与剧情咬合

| 秘密冲突 | 金手指来源成谜 | 不知道为什么会获得这个能力 |
| 社会冲突 | 能力引来觊觎 | 别人想要这个能力 |

## 五、能力展示节奏

### 5.1 能力展示的方法

**【目标】**：让能力展示有节奏感，不重复、不无聊

**【方法】**：能力展示类型

| 展示类型 | 描述 | 适用时机 |
|---------|------|---------|
| **初秀** | 第一次展示能力 | 开篇/早期 |
| **困境** | 用能力解决难题 | 危机时刻 |
| **升级** | 能力进阶/新能力 | 突破节点 |
| **限制** | 展示能力的边界 | 遭遇克制 |
| **对抗** | 能力对能力 | 高手对决 |
| **代价** | 展示使用代价 | 关键选择 |

```

| 困境 | 第X章 | [描述] | [达到什么效果] |

| 限制 | 第X章 | [描述] | [达到什么效果] |
| 对抗 | 第X章 | [描述] | [达到什么效果] |
| 代价 | 第X章 | [描述] | [达到什么效果] |

【节奏检查】
- [ ] 展示是否避免重复？
- [ ] 每次展示是否有新意？
- [ ] 展示是否推动剧情？
```

### 5.2 避免能力展示无聊

**【常见问题】**：
- 每次都用同样方式展示
- 能力展示没有新意
- 读者可以预测能力发展

**【解决方案】**：
1. 用不同场景展示
2. 用对抗展示能力边界
3. 用代价展示能力代价
4. 用组合展示能力变化

## 六、战斗/冲突设计

```

1. 参战方：
   - 主角方：能力/状态/弱点
   - 敌方：能力/状态/弱点

2. 战斗目的：
   - 为什么要打
   - 胜利条件是什么
   - 失败后果是什么

3. 战斗限制：
   - 环境限制
   - 能力限制
   - 时间限制

4. 战斗转折：
   - 第一次转折：[描述]
   - 第二次转折：[描述]
   - 最终结果：[描述]
```

- 转折：药老力量觉醒
- 代价：打完之后如何

## 八、常见问题与修复

### 问题1：战力崩坏

**表现**：主角时强时弱，战力忽高忽低

**修复方法**：
1. 建立战力等级表
2. 严格控制越级挑战
3. 设置实力评估场景

### 问题2：金手指太bug

**表现**：主角没有挑战，对手太弱

**修复方法**：
1. 增加金手指限制
2. 增加金手指代价
3. 增加金手指的副作用

### 问题3：能力展示重复

**表现**：每次都是同样的方式展示能力

**修复方法**：
1. 用不同场景展示
2. 用对抗展示能力
3. 用代价展示能力

### 问题4：战斗无聊

**表现**：战斗只是比谁力量大

**修复方法**：
1. 增加战术元素
2. 利用环境
3. 利用能力相克

### 问题5：升级太快

**表现**：主角升级速度太快，没有成长感

**修复方法**：
1. 增加升级难度
2. 增加升级资源稀缺性
3. 增加升级瓶颈
