# 世界观设定指南

> 本指南对应评分体系"世界观与设定"维度。

---

## 一、核心原则

1. **自洽是底线**：设定内部必须逻辑一致，矛盾会摧毁读者信任
2. **设定即剧情**：设定必须驱动故事，而非只是装饰背景
3. **逐层展开**：用"冰山理论"——你知道90%，只展示10%
4. **质感是灵魂**：科幻要可信，魔法要有逻辑，玄幻要有独特韵律
5. **创新要找角度**：不是颠覆一切，而是在熟悉中找到新切入点

---

## 二、世界观构建方法论

### 2.1 从核心假设出发

**【目标】**：用一个核心假设推演出整个世界观

**【方法】**：核心假设 → 必然推论 → 细节填充

```
世界观构建公式：
核心假设（1个）→ 连锁反应（3-5个）→ 世界呈现

核心假设例子：
- 如果时间可以倒流但只能回到自己曾经过的地方……
- 如果魔法需要用生命代价换取……
- 如果AI可以完美模拟任何人类……
```

**【正例特征】**（《三体》黑暗森林）：
- 核心假设：宇宙资源有限 + 文明必然扩张 + 猜疑链 → 黑暗森林法则
- 从一个假设推演出整个宇宙文明图景

**【AI指令模板】**：

```
请用核心假设法构建世界观：

【第一步：定义核心假设】
这个世界最核心的规则/限制是什么？
（必须是1句话，可以是"如果……那么……"句式）

核心假设：[描述]

【第二步：推演连锁反应】
这个假设会导致什么？（列出5-8个推论）
推论1：[描述] → 影响：[日常/社会/政治/军事/文化]
推论2：[同上]
……

【第三步：选择3个最重要的推论】
1. [选择并详细化]
2. [选择并详细化]
3. [选择并详细化]

【第四步：设计细节】
- 日常生活的变化：[描述]
- 社会结构的变化：[描述]
- 核心冲突的可能：[描述]

【第五步：验证自洽性】
- 假设1 + 假设2 + 假设3 是否自洽？
- 有无矛盾需要修正？
```

### 2.2 世界观的层级结构

**【目标】**：让世界观有深度和层次感

**【方法】**：三层世界构建法

```
世界观三层结构：

┌─────────────────────────────────────────┐
│ 第一层：可见世界（10%呈现给读者）          │
│ - 表层设定：直接可见的规则、能力、社会      │
│ - 日常运作：读者角色生活其中的世界          │
├─────────────────────────────────────────┤
│ 第二层：深层规则（30%通过剧情揭示）         │
│ - 隐藏机制：驱动世界运作的底层规则          │
│ - 历史脉络：如何形成现在的世界             │
│ - 势力格局：各方力量的博弈                 │
├─────────────────────────────────────────┤
│ 第三层：终极秘密（60%作者知道）             │
│ - 世界起源：一切的起点                     │
│ - 终极真相：最终boss/危机的根源            │
│ - 未解之谜：留给读者的思考空间             │
└─────────────────────────────────────────┘
```

**【AI指令模板】**：

```
请构建三层世界观：

【第一层：可见世界】
- 地理：[描述主要地区]
- 社会：[描述社会结构]
- 日常：[普通人如何生活]
- 能力系统：[如适用，简要描述]

【第二层：深层规则】
- 隐藏机制：[2-3个不为人知的规则]
- 历史脉络：[世界如何变成现在这样，3-5个关键事件]
- 势力格局：[列出主要势力及关系]

【第三层：终极秘密】
- 世界起源：[描述]
- 终极真相：[描述]
- 留给读者的谜题：[1-2个]

【各层展示计划】
| 层级 | 展示方式 | 展示时机 |
|------|---------|---------|
| 第一层 | 直接描写 | 开篇 |
| 第二层 | 剧情揭示 | 中段 |
| 第三层 | 高潮/结局 | 后半段 |
```

## 三、自洽性检查法

### 3.1 自洽性检查清单

**【目标】**：确保设定内部逻辑一致

**【检查维度】**：

| 检查维度 | 检查问题 | 常见错误 |
|---------|---------|---------|
| **能力边界** | 这个能力能做什么？不能做什么？ | 主角能力随需要伸缩 |
| **社会逻辑** | 社会如何运转？阶层如何形成？ | 贵族和平民生活无区别 |
| **历史逻辑** | 历史事件如何导致现在？ | 历史与现实脱节 |
| **地理逻辑** | 地理位置如何影响事件？ | 沙漠出现热带雨林 |
| **战力逻辑** | 谁强谁弱？为什么？ | 战力忽高忽低 |

**【AI指令模板】**：

```
请检查以下设定的自洽性：

【设定内容】
[粘贴设定]

【自洽性检查】
请逐一检查：

1. 能力边界检查
   - 能力A的上限：[描述]
   - 能力A的代价：[描述]
   - 能力A的例外：[描述]
   → 是否自洽：[是/否，如否则修改]

2. 社会逻辑检查
   - 阶层A如何获取资源：[描述]
   - 阶层B如何获取资源：[描述]
   → 资源分配逻辑是否矛盾：[是/否]

3. 历史逻辑检查
   - 历史事件C导致：[描述]
   - 历史事件D导致：[描述]
   → 事件C和D是否矛盾：[是/否]

4. 战力逻辑检查
   - 角色A vs 角色B：[谁更强]
   - 理由：[描述]
   → 战力对比是否合理：[是/否]

【发现问题】
列出所有发现的不自洽之处，并给出修改建议。
```

### 3.2 常见自洽性错误与修复

| 错误类型 | 表现 | 修复方法 |
|---------|------|---------|
| 战力崩坏 | 主角时强时弱 | 建立战力等级表，严格遵守 |
| 规则反复 | 设定被随意改变 | 在设定文档中明确"不可违背"项 |
| 地理混乱 | 位置关系矛盾 | 制作地图，标注距离和时间 |
| 社会割裂 | 社会阶层与生活脱节 | 思考资源分配逻辑 |
| 时间混乱 | 事件先后矛盾 | 建立时间线 |

## 四、设定-剧情咬合设计

### 4.1 让设定驱动剧情

**【目标】**：设定不是背景板，而是剧情的发动机

**【方法】**：设定-剧情双向设计

```
咬合设计公式：
设定 × 冲突 = 剧情

冲突来源：
- 设定规则 vs 角色欲望
- 设定限制 vs 角色能力
- 设定秘密 vs 角色认知
```

**【正例特征】**（《基地》心理史学）：
- 心理史学不是背景，而是故事的核心驱动力
- 所有剧情都围绕这个设定展开

**【反例特征】**：
- 玄幻世界很大，但故事只发生在"一个普通小镇"
- 魔法很强，但主角大部分时间在打普通人
- 科技很发达，但社会结构和古代一模一样

**【AI指令模板】**：

```
请设计设定-剧情咬合：

【世界观设定】
[描述核心世界观]

【核心冲突】
[描述主线冲突]

【咬合设计】
请为每个主要剧情节点设计设定驱动：

剧情节点1：[描述]
- 驱动的设定：[设定A]
- 冲突来源：[设定A如何产生冲突]
- 角色应对：[主角如何使用/对抗设定]

剧情节点2：[同上]

剧情节点3：[同上]

【检查】
- [ ] 设定是否在每个主要剧情中都发挥作用？
- [ ] 是否存在"设定装饰"（设定存在但不影响剧情）？
- [ ] 修改任意设定，剧情是否会随之改变？
```

### 4.2 设定驱动冲突的类型

| 驱动类型 | 描述 | 举例 |
|---------|------|------|
| **规则冲突** | 设定规则与角色欲望对立 | 想修炼但资质被废 |
| **限制冲突** | 能力有边界，过不去 | 有钱但买不到解药 |
| **秘密冲突** | 角色不知道的设定秘密 | 以为是好人其实是敌人 |
| **代价冲突** | 使用能力需要付出代价 | 魔法需要生命 |
| **逻辑冲突** | 利用设定规则制造困境 | 必须遵守诺言但诺言导致毁灭 |

## 五、设定展开节奏

### 5.1 避免信息倾倒

**【目标】**：设定信息自然融入，不让读者感觉在读"说明书"

**【方法】**：场景化展示法

```
设定展示三原则：
1. 通过角色行动展示（做什么比说什么更重要）
2. 通过解决问题展示（遇到问题时才揭示规则）
3. 通过对比展示（普通vs特殊，揭示差异）
```

**【正例特征】**（《冰与火之歌》）：
- 世界观通过碎片信息逐渐呈现
- 读者像考古学家一样逐渐发现全貌

**【反例特征】**：
- 开头用大段文字介绍世界观
- 角色A向角色B解释"你应该知道的事"
- 用脚注或括号解释设定

**【AI指令模板】**：

```
请用场景化方式展示以下设定：

【需要展示的设定】
[描述]

【场景化方案】
请设计一个场景来展示这个设定：

场景设计：
- 场景：[描述]
- 角色：[谁在场]
- 目标：[角色想做什么]
- 冲突：[设定如何制造冲突]
- 展示方式：[通过角色行动/对话/观察]

【避免项检查】
- [ ] 是否避免了"说明书式"介绍？
- [ ] 是否避免了角色互相对话解释设定？
- [ ] 是否避免了括号内解释？
- [ ] 读者是否需要先知道这个设定才能理解场景？
```

### 5.2 设定展示的时机选择

| 时机 | 适用情况 | 效果 |
|------|---------|------|
| **需要时** | 读者/角色遇到相关问题时 | 最自然，但可能显得"刚好知道" |
| **好奇时** | 读者刚产生疑问时 | 满足好奇心，但需要精确把握时机 |
| **冲突时** | 设定被违反或挑战时 | 最具冲击力，但需要铺垫 |
| **日常时** | 角色正常生活时 | 自然，但可能平淡 |

**【AI指令模板】**：

```
请规划设定展示时机：

【设定清单】
需要展示的设定：[列出]

【展示时机规划】
| 设定 | 展示时机 | 展示方式 | 为什么此时展示 |
|------|---------|---------|--------------|
| 设定A | 第X章 | [方式] | [原因] |
| 设定B | 第X章 | [方式] | [原因] |

【优先级】
必须尽早展示（否则影响理解）：[列出]
可以延后展示：[列出]
可以省略展示（让读者推测）：[列出]
```

## 六、技术/魔法质感写法

### 6.1 科幻技术质感

**【目标】**：让科幻设定有可信度，即使幻想也有科学基础

**【方法】**：硬科幻三层法

```
科幻质感三要素：
1. 物理基础：基于已知物理的推演
2. 技术细节：让读者看到"怎么做"
3. 社会影响：技术如何改变世界
```

**【正例特征】**（《三体》水滴）：
- 基于物理原理的推演
- 技术细节让读者相信
- 与人类科技形成对比

**【反例特征】**：
- 违反基础物理没有解释
- 技术是"魔法"——想用就用
- 技术对社会毫无影响

**【AI指令模板】**：

```
【技术细节】

```

### 6.2 魔法/玄幻体系质感

**【目标】**：让魔法有逻辑，让修炼有层次

**【方法】**：魔法体系设计公式

```
魔法体系 = 力量来源 + 运用方式 + 代价限制

- 力量来源：魔力/灵力/元素/意志力……
- 运用方式：咒语/阵法/契约/天赋……
- 代价限制：消耗/代价/限制/禁忌……
```

**【正例特征】**（《诡秘之主》）：
- 22条神之途径，路径清晰
- 每个序列有明确的能力和代价
- 扮演法是独特的修炼方式

**【反例特征】**：
- 魔法是万能的，什么问题都能解决
- 等级体系随意变化
- 没有代价，想用就用

**【AI指令模板】**：

## 七、创新方法

### 7.1 在老题材中找新角度

**【目标】**：不是颠覆一切，而是在熟悉中找独特

**【方法】**：旧瓶装新酒

```
创新方向：
1. 视角创新：从不同角色/角度切入
2. 规则创新：改变一个基础规则
3. 主题创新：用旧题材探讨新问题
4. 组合创新：两种以上元素的新组合
```

**【正例特征】**（《放开那个女巫》）：
- 穿越+种田+西幻，不是新元素
- 但用科学改造女巫世界是新角度
- 技术与魔法的结合是独特设计

## 八、常见问题与修复

### 问题1：设定太复杂

**表现**：读者记不住，叙事混乱

**修复方法**：
1. 优先展示最核心的设定
2. 其他设定在需要时才揭示
3. 建立"设定展示优先级"

### 问题2：设定太简单

**表现**：世界没有深度，像平面背景

**修复方法**：
1. 增加隐藏设定
2. 增加历史脉络
3. 增加势力格局

### 问题3：设定和剧情脱节

**表现**：设定存在但不影响故事

**修复方法**：
1. 重新设计"设定驱动冲突"
2. 检查每个主要剧情是否有设定参与
3. 删除不影响剧情的"装饰设定"

### 问题4：设定解释太生硬

**表现**：突然插入大段设定说明

**修复方法**：
1. 改为场景化展示
2. 改为"做中学"
3. 改为对比展示
