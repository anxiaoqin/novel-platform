# 角色设计指南

> 本指南对应评分体系"角色与人物"维度。

---

## 一、核心原则

1. **辨识度是生命**：主角必须是"只有这一个"，而非"可以换成任何人"
2. **功能是底线**：每个角色必须有存在的意义，删除他故事会不同
3. **对话即性格**：读者仅凭对话就能辨认说话者
4. **反派是镜子**：好反派让主角更有光彩
5. **群像有主次**：配角出彩但不失焦，主角永远在舞台中央

---

## 二、主角设计方法

### 2.1 辨识度构建法

**【目标】**：让主角独一无二，读者永远不会忘记

**【方法】**：三维辨识度设计

```
主角辨识度 = 外在辨识（1）+ 行为辨识（2）+ 内在辨识（3）

1. 外在辨识：
   - 独特的外貌/装扮
   - 独特的说话方式
   - 独特的习惯动作

2. 行为辨识：
   - 面对冲突时的独特反应
   - 做决定时的独特逻辑
   - 与人互动时的独特方式

3. 内在辨识：
   - 独特的欲望/恐惧
   - 独特的价值观/底线
   - 独特的成长障碍
```

**【正例特征】**：
- 韦小宝：痞气+义气+贪财，讲义气但又耍小聪明
- 孙悟空：反叛+智慧+骄傲，"齐天大圣"的傲骨
- 克莱恩：谨慎+责任+挣扎，"穷竭智略"的小人物

**【反例特征】**：
- 主角没有明确缺点
- 主角可以用"普通人"模板预测
- 所有网文主角都是"废柴逆袭"

**【AI指令模板】**：

```
请为主角设计辨识度：

【基础信息】
- 姓名：[名字要有独特性]
- 身份：[职业/地位]

【外在辨识】
- 外貌：[描述独特的1-2个特征]
- 穿着：[描述标志性装扮]
- 说话方式：[语气/用词/习惯]
- 口头禅：[如有，1-2句即可]

【行为辨识】
- 面对危险时：[描述反应]
- 面对选择时：[描述决策方式]
- 与陌生人互动：[描述方式]
- 独处时：[描述习惯]

【内在辨识】
- 最想要什么：[描述]
- 最害怕什么：[描述]
- 绝对不能做的事：[底线]
- 与其他主角最大的不同：[描述]

【辨识度检查】
- [ ] 读者能否仅凭3个特征认出主角？
- [ ] 主角的行为是否独一无二？
- [ ] 主角可以被替代吗？
```

### 2.2 内心弧线设计

**【目标】**：让主角在故事中成长或堕落，变化可信且有层次

**【方法】**：弧线设计四阶段

```
内心弧线 = 默认值 → 动摇 → 危机 → 新值

1. 默认值（故事开始时）：
   - 性格特征
   - 核心信念
   - 行为模式

2. 动摇（冲突出现时）：
   - 什么挑战了默认值
   - 主角如何应对
   - 内心开始出现裂缝

3. 危机（核心冲突时）：
   - 默认值彻底崩溃的时刻
   - 主角被迫做出选择
   - 变化在此刻发生

4. 新值（故事结尾时）：
   - 新的性格特征
   - 新的核心信念
   - 新的行为模式
```

**【正例特征】**（《活着》福贵）：
- 起点：纨绔子弟
- 动摇：家产输光
- 危机：所有亲人去世
- 新值：活着本身就是意义

**【AI指令模板】**：

```
请为主角设计内心弧线：

【起点状态】
- 核心信念：[主角相信什么]
- 价值观：[主角看重什么]
- 行为模式：[通常如何行动]
- 缺陷/弱点：[这个信念导致的缺点]

【动摇阶段】
- 触发事件1：[描述] → 内心变化：[描述]
- 触发事件2：[描述] → 内心变化：[描述]
- 动摇表现：[主角开始质疑自己]

【危机阶段】
- 核心事件：[描述]
- 默认值崩溃时刻：[描述]
- 主角的抉择：[描述]
- 变化发生：[从X变为Y]

【新状态】
- 新的核心信念：[描述]
- 新的价值观：[描述]
- 新的行为模式：[描述]
- 这个变化如何体现：[象征/动作/决定]

【弧线检查】
- [ ] 变化是由事件驱动而非强行安排
- [ ] 有足够的铺垫让变化可信
- [ ] 变化是不可逆的
- [ ] 变化符合人物性格逻辑
```

### 2.3 动机系统设计

**【目标】**：让主角的行动有内在驱动力，不是被推着走

**【方法】**：动机层次设计

```
动机层次（从表层到深层）：
表层动机：当下想要什么
↓ 为什么想要
中层动机：角色需要证明什么
↓ 根源
深层动机：角色的核心恐惧/渴望

```

**【AI指令模板】**：

```
请设计主角的动机系统：

【表层动机】
- 目标：[主角当下想要什么]
- 行动：[为达到目标做了什么]

【中层动机】
- 想要证明：[主角需要证明什么]
- 为什么要证明：[起源故事]

【深层动机】
- 核心渴望：[最深处想要什么]
- 核心恐惧：[最害怕什么]
- 渴望与恐惧的关系：[如何相互作用]

【动机可视化】
请用一个场景展示深层动机：
- 场景：[描述]
- 主角的渴望如何体现：[描述]
- 恐惧如何被触发：[描述]

【动机一致性检查】
- [ ] 主角的行动是否始终服务于动机？
- [ ] 动机是否足够强烈支撑主线？
- [ ] 动机的变化是否有迹可循？
```

## 三、配角设计方法

### 3.1 功能定位法

**【目标】**：每个角色都有不可替代的功能

**【方法】**：角色功能分类

```
配角功能类型：

┌─────────────────────────────────────────┐
│ 镜像配角                                  │
│ 功能：通过对比/相似揭示主角               │
│ 特点：与主角有相似处境但不同选择          │
├─────────────────────────────────────────┤
│ 导师配角                                  │
│ 功能：推动主角成长                        │
│ 特点：提供知识/技能/指引                 │
├─────────────────────────────────────────┤
│ 盟友配角                                  │
│ 功能：提供支持/资源/情感陪伴              │
│ 特点：与主角有共同目标                    │
├─────────────────────────────────────────┤
│ 对手配角                                  │
│ 功能：制造冲突/推动情节                   │
│ 特点：阻碍主角但不是反派                  │
├─────────────────────────────────────────┤
│ 见证配角                                  │
│ 功能：旁观者视角/情感共鸣                 │
│ 特点：读者通过他理解主角                  │
└─────────────────────────────────────────┘
```

**【正例特征】**（《射雕英雄传》郭靖配）：
- 周伯通：镜像（天真相近）+导师（教授武功）
- 黄蓉：盟友+镜像（聪明对比）
- 欧阳锋：对手+反派（宿敌）

**【AI指令模板】**：

```
请设计主要配角：

【配角1】
- 姓名：[名字]
- 功能类型：[镜像/导师/盟友/对手/见证/其他]
- 与主角的关系：[描述]

- 功能具体化：
  如何服务于主角/故事：[描述]
  删除这个角色会有什么影响：[描述]

- 人物设计：
  外在：[描述]
  内在：[描述]
  独特之处：[描述]

- 弧线设计：
  起点：[描述]
  变化：[描述]
  终点：[描述]

【配角2】：[同上]
【配角3】：[同上]

【功能检查】
- [ ] 每个配角都有明确功能？
- [ ] 功能是否重叠？
- [ ] 是否有"酱油角色"（存在但无意义）？
```

### 3.2 避免功能重叠

**【目标】**：每个配角都是独特的，不存在"可以替换"的配角

**【检查方法】**：

| 问题 | 表现 | 修复 |
|------|------|------|
| 功能完全相同 | 两个配角都"帮助主角" | 差异化：一个是情感支持，一个是实际帮助 |
| 性格完全相同 | 两个配角说话方式一样 | 差异化：不同的背景、不同的说话风格 |
| 作用完全相同 | 两个配角都"死在关键时刻" | 差异化：不同死法、不同意义 |

## 四、对话设计方法

### 4.1 性格化对话法

**【目标】**：读者仅凭对话就能辨认说话者

**【方法】**：对话四要素设计

```
对话性格化 = 词汇选择 + 句式特征 + 话题偏好 + 潜台词风格

1. 词汇选择：
   - 教育背景影响用词
   - 地域影响用词
   - 职业影响术语

2. 句式特征：
   - 说话长短
   - 疑问句/祈使句使用
   - 是否犹豫/打断

3. 话题偏好：
   - 关心什么话题
   - 回避什么话题
   - 习惯从什么角度切入

4. 潜台词风格：
   - 直接vs委婉
   - 讽刺vs真诚
   - 暗示vs明说
```

**【正例特征】**（《鹿鼎记》）：
- 韦小宝：市井口语、"老子"、狡猾油滑
- 康熙：书面语少、"朕"、威严但亲切
- 海大富：慢条斯理、直指要害

**【AI指令模板】**：

```
请为以下角色设计性格化对话：

【角色信息】
- 姓名：[名字]
- 背景：[职业/教育/地域]
- 性格：[核心性格]

【对话设计】
- 词汇选择：[特有的词汇/用语/口头禅]
- 句式特征：[长短/停顿/语气]
- 话题偏好：[常说/少说/回避]
- 潜台词风格：[直接/暗示/讽刺]

【对比设计】
如果两个角色对话，请设计对比：
角色A的说话方式：[描述]
角色B的说话方式：[描述]
对话中的差异如何体现：[描述]

【检验】
请写一段这两个角色的对话：
[要求：读者不看名字也能分辨谁是谁]
```

### 4.2 潜台词写法

**【目标】**：让对话有"言外之意"，增加深度

**【方法】**：冰山理论

```
潜台词公式：
表面说的话 ≠ 实际想表达

潜台词类型：
1. 直接掩盖：说A实际是B
2. 转移话题：用A话题掩盖真正想说的B
3. 试探性：看似问A实际在问B
4. 双关：表面一个意思，实际另一个意思
```

**【正例特征】**：
- 角色说"今天天气真好"，实际在说"我心情不好"
- 角色问"你还记得她吗"，实际在问"你还记得我们的约定吗"

**【AI指令模板】**：

```
请设计有潜台词的对话：

【场景】
- 地点：[描述]
- 双方：[角色A和角色B]
- 背景：[为什么这段对话重要]

【表面对话】
角色A说的话：[粘贴]
角色B说的话：[粘贴]

【潜台词分析】
角色A实际想表达：[描述]
角色B实际想表达：[描述]

【潜台词实现方式】
- 表面内容：[描述]
- 暗示方式：[动作/语气/眼神]
- 读者如何感知：[描述]

【检验】
这段对话是否做到了：
- [ ] 表面和潜台词不同？
- [ ] 潜台词能让读者感受到？
- [ ] 不需要旁白解释？
```

### 4.3 对话推动法

**【目标】**：让每个对话都有存在的意义

**【方法】**：对话功能检查

```
对话必须至少满足一个：
1. 推动剧情：对话后事情必须发生变化
2. 揭示性格：通过对话让读者更了解角色
3. 建立关系：改变角色之间的关系
4. 制造冲突：对话引发或加剧冲突

禁止的对话：
- 无意义的寒暄（除非有功能）
- 信息重复（前面说了后面又说）
- 可以用动作代替的话
```

**【AI指令模板】**：

```
请检查以下对话是否有意义：

【对话内容】
[粘贴对话]

【功能分析】
这段对话的功能：
- [ ] 推动剧情：变化了什么？
- [ ] 揭示性格：揭示了什么？
- [ ] 建立关系：关系如何变化？
- [ ] 制造冲突：冲突是什么？

如果都不满足：
- 如何修改让它有功能？

【删除测试】
- 如果删除这段对话，故事会受影响吗？
- 如果不受影响，是否应该删除？
```

## 五、反派/对手设计方法

### 5.1 逻辑化反派

**【目标】**：让反派相信自己是正义的，让读者理解反派

**【方法】**：反派三要素

```
反派设计 = 合理动机 + 合理逻辑 + 合理手段

1. 合理动机：
   - 反派真正想要什么？
   - 为什么想要？
   - 这个欲望是否让读者同情？

2. 合理逻辑：
   - 反派的世界观是什么？
   - 基于这个世界观，他的行动是否合理？
   - 他的逻辑是否能让读者"理解"？

3. 合理手段：
   - 他的方法是否与性格一致？
   - 他是否有底线？
   - 他是否会在必要时妥协？
```

**【正例特征】**（《猎人》蚁王）：
- 动机：想要理解人类
- 逻辑：强者生存
- 复杂性：即使作为反派也有魅力

**【反例特征】**：
- 反派动机是"我就是想毁灭世界"
- 反派为坏而坏，没有深层原因
- 反派比主角还弱智

**【AI指令模板】**：

```
请设计一个逻辑化的反派：

【反派基础】
- 姓名：[名字]
- 身份：[地位/职业]

【动机设计】
- 真正想要什么：[描述]
- 为什么想要：[起源故事]
- 这个欲望的合理性：[描述]

【逻辑设计】
- 世界观：[描述反派相信什么]
- 行动逻辑：[基于这个世界观他如何行动]
- 与主角的对比：[主角vs反派的核心分歧]

【手段设计】
- 他会做什么：[描述]
- 他的底线：[什么是他不会做的]
- 他的弱点：[什么可以击败他]

【与主角的关系】
- 初始关系：[描述]
- 冲突焦点：[什么让他们对立]
- 终极对决时：[什么选择摆在面前]

【检验】
- [ ] 读者能理解反派的动机吗？
- [ ] 反派的行动是否有一致性？
- [ ] 反派是否可以被击败而非被"消灭"？
```

### 5.2 反派塑造层次

| 层次 | 描述 | 举例 |
|------|------|------|
| **工具反派** | 没有动机，只是阻碍 | 灾难、机械敌人 |
| **性格反派** | 有性格但动机简单 | 贪婪、嫉妒 |
| **逻辑反派** | 有完整逻辑，但与主角对立 | 不同价值观 |
| **镜像反派** | 与主角相似但选择不同 | 另一个可能的自己 |
| **哲理反派** | 代表一种思想，与主角的思想对决 | 命题对立 |

## 六、群像平衡方法

### 6.1 戏份分配原则

**【目标】**：主角永远是核心，配角各有亮点但不失焦

**【戏份分配表】**：

| 角色类型 | 戏份比例 | 说明 |
|---------|---------|------|
| 主角 | 50%+ | 永远是中心 |
| 主要配角 | 20-30% | 服务于主角和主线 |
| 次要配角 | 10-20% | 有功能但不出风头 |
| 工具角色 | <10% | 完成功能即退场 |

### 6.2 避免失衡的方法

**【问题表现】**：
- 配角比主角更有魅力
- 配角背景故事比主角还详细
- 主角变成"解说员"

**【修复方法】**：
1. 限制配角章节数
2. 配角的每次出场都服务于主角弧线
3. 配角的背景故事通过主角视角展开

**【AI指令模板】**：

```
请设计群像戏份分配：

【角色清单】
1. 主角：[名字] → 戏份：50%+
2. 主要配角：[名字] → 戏份：20-30%
3. 主要配角：[名字] → 戏份：20-30%
4. 次要配角：[名字] → 戏份：10-20%
5. 次要配角：[名字] → 戏份：10-20%
6. 工具角色：[名字] → 戏份：<10%

【各角色出场规划】
主角：[全程]
配角A：第X章首次出场，共出现X次
配角B：[同上]

【平衡检查】
- [ ] 主角戏份是否占主导？
- [ ] 配角是否有喧宾夺主的风险？
- [ ] 每个配角的出场是否都有意义？
```

## 七、角色关系网设计

### 7.1 关系网构建

**【目标】**：让角色关系形成网络而非单线

**【方法】**：关系网矩阵

```
关系网设计：
         主角    配角A    配角B    反派
主角      -      X关系    Y关系    Z关系
```

## 八、常见问题与修复

### 问题1：主角太脸谱化

**表现**：换一个名字就是另一个小说的主角

**修复方法**：
1. 增加独特的缺陷/弱点
2. 增加独特的行为模式
3. 增加独特的说话方式

### 问题2：对话太生硬

**表现**：所有角色说话方式一样

**修复方法**：
1. 为每个角色设计独特的说话方式
2. 通过对比让差异更明显
3. 让对话有潜台词

### 问题3：反派太弱

**表现**：反派只是为了存在而存在

**修复方法**：
1. 给反派合理的动机
2. 让反派相信自己是正义的
3. 让反派给主角制造真正的困难

### 问题4：群像失衡

**表现**：配角比主角还出彩

**修复方法**：
1. 限制配角戏份
2. 确保配角每次出场都服务于主角
3. 减少配角独立故事线

### 问题5：角色OOC

**表现**：角色性格在故事中突然变化

**修复方法**：
1. 在变化前增加铺垫
2. 确保变化符合角色逻辑
3. 让读者看到变化的过程
