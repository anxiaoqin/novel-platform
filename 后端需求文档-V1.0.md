# 十方天地AI网文创作平台 — 后端需求文档 V1.0

> 基于 UI 设计规范 V1.0，梳理前端无法独立完成、需后端配合的接口与功能

---

## 一、需新建的 API

### 1. 角色/设定库 CRUD

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 获取角色列表 | GET | `/novels/{novel_id}/characters` | 支持分页、按分类筛选（角色/势力/功法/物品/地理/等级体系） |
| 创建角色/设定 | POST | `/novels/{novel_id}/characters` | body: { name, category, identity, appearance, personality, power_level, bio } |
| 获取角色详情 | GET | `/characters/{id}` | 返回完整信息含关系、成长线、经典台词 |
| 更新角色/设定 | PUT | `/characters/{id}` | 同创建字段 |
| 删除角色/设定 | DELETE | `/characters/{id}` | 软删除 |
| 获取角色关系 | GET | `/characters/{id}/relations` | 返回 [{target_id, target_name, relation_type}] |
| 添加角色关系 | POST | `/characters/{id}/relations` | body: { target_id, relation_type } |
| 删除角色关系 | DELETE | `/characters/{id}/relations/{target_id}` | |
| 角色关系图谱 | GET | `/novels/{novel_id}/characters/graph` | 返回力导向图数据：{ nodes: [{id,name,category}], edges: [{source,target,type}] } |

**数据模型建议：**
```sql
CREATE TABLE characters (
  id SERIAL PRIMARY KEY,
  novel_id INTEGER REFERENCES novels(id) NOT NULL,
  name VARCHAR(100) NOT NULL,
  category VARCHAR(20) NOT NULL,  -- character/faction/skill/item/location/level_system
  identity VARCHAR(200),
  appearance TEXT,
  personality TEXT,
  power_level VARCHAR(50),
  bio TEXT,
  avatar_url VARCHAR(500),
  ai_auto_reference BOOLEAN DEFAULT false,  -- AI生成时是否自动调用此设定
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE character_relations (
  id SERIAL PRIMARY KEY,
  character_id INTEGER REFERENCES characters(id) ON DELETE CASCADE,
  target_id INTEGER REFERENCES characters(id) ON DELETE CASCADE,
  relation_type VARCHAR(50),  -- enemy/friend/mentor/relative
  UNIQUE(character_id, target_id)
);

CREATE TABLE character_growth (
  id SERIAL PRIMARY KEY,
  character_id INTEGER REFERENCES characters(id) ON DELETE CASCADE,
  time_point VARCHAR(200),  -- 时间节点描述
  key_event TEXT,            -- 关键事件
  sort_order INTEGER DEFAULT 0
);

CREATE TABLE character_quotes (
  id SERIAL PRIMARY KEY,
  character_id INTEGER REFERENCES characters(id) ON DELETE CASCADE,
  quote TEXT NOT NULL,
  context VARCHAR(500)
);
```

### 2. 剧情时间线

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 获取时间线事件列表 | GET | `/novels/{novel_id}/timeline` | 支持 type 筛选（主线/角色支线/世界观事件），支持按剧情时间/创作章节排序 |
| 创建时间线事件 | POST | `/novels/{novel_id}/timeline` | body: { name, time_point, description, event_type, related_characters[], outline_chapter_id } |
| 更新时间线事件 | PUT | `/timeline/{id}` | 同创建字段 |
| 删除时间线事件 | DELETE | `/timeline/{id}` | |
| 时间线校验 | POST | `/novels/{novel_id}/timeline/validate` | 返回时间冲突列表：[{event_a_id, event_b_id, conflict_reason}] |
| 关联大纲章节 | POST | `/timeline/{id}/link-outline` | body: { outline_node_id } |

**数据模型建议：**
```sql
CREATE TABLE timeline_events (
  id SERIAL PRIMARY KEY,
  novel_id INTEGER REFERENCES novels(id) NOT NULL,
  name VARCHAR(200) NOT NULL,
  time_point VARCHAR(200),        -- 剧情内时间描述
  chapter_index INTEGER,           -- 对应创作章节序号
  description TEXT,
  event_type VARCHAR(20) NOT NULL, -- main_plot/character_subplot/world_event
  related_characters JSONB,        -- [character_id, ...]
  outline_node_id INTEGER,         -- 关联大纲章节
  sort_order INTEGER DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW()
);
```

## 二、需修改的现有 API

### 1. analyze-paragraph 增强
- **现状：** 返回 suggestions + extracted
- **需增加：** 每条 suggestion 增加 `type` 枚举值（铺垫/冲突/反转/高潮/过渡），对应前端功能色标签
- **需增加：** `granularity` 参数（chapter/scene/paragraph），控制事件颗粒度

### 2. save-outline-events 增强
- **现状：** 扁平事件列表
- **需增加：** 支持多级树形结构（卷-章-节-场景），每节点有 parent_id + order

### 3. novels 列表增强
- **现状：** 返回 items[]
- **需增加：** 每个作品增加 `genre`（题材标签）、`cover_url`（封面）、`last_updated`（最近更新时间）

### 4. chapters 列表增强
- **现状：** 基础列表
- **需增加：** 返回每个章节的 `word_count`、`updated_at`，支持按更新时间排序

## 五、接口通用规范

1. **响应格式统一：**
```json
{
  "success": true,
  "data": {},
  "error": null
}
```

2. **分页统一：** `?page=1&page_size=20`，返回 `{ items: [], total: N, page: N, page_size: N }`

3. **认证：** 所有接口需 Authorization: Bearer {token}

4. **AI 接口流式支持：** 续写/重写/扩写/填充等生成类接口建议支持 SSE 流式返回，前端逐字输出模拟打字效果
